Selects the next action.

Parameters

  • predictor – RLPredictor/GymPredictor object whose policy to follow. If set to None, use a random policy.

  • next_state – State to evaluate predictor’s policy on.

  • test – Whether or not to bypass exploration (if predictor is not None). For discrete action problems, the exploration policy is epsilon-greedy. For continuous action problems, the exploration is achieved by adding noise to action outputs.

  • state_preprocessor – State preprocessor to use to preprocess states
